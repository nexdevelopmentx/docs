## Requirements

* FiveM server
* MySQL or MariaDB
* One of: [`qbx_core`](https://github.com/Qbox-project/qbx_core), [`qb-core`](https://github.com/qbcore-framework/qb-core), or [`es_extended`](https://github.com/esx-framework/esx_core)
* [`oxmysql`](https://github.com/overextended/oxmysql) and [`ox_lib`](https://github.com/overextended/ox_lib)

## Steps

1. Drop the `nex_garages` folder into `resources/[nex]/`.

2. Import the SQL file for your framework into your database:
   * **QBCore / Qbox** → `sql/qb.sql`
   * **ESX** → `sql/esx.sql`

   <Warning>
     This step is required. The SQL adds the columns the script needs on your existing vehicle tables. Skipping it will cause database errors.
   </Warning>

3. Open `config.lua` and set:
   * `Config.Framework` — leave on `"auto"` or pick `"QBCore"`, `"Qbox"`, or `"ESX"`
   * `Config.FuelSystem` — match your fuel script (or `"none"`)
   * `Config.VehicleKeys` — match your keys script (or `"none"`)
   * `Config.DrawText` — match your TextUI resource

4. Add it to `server.cfg` **after** your framework, oxmysql, and ox_lib:

   ```
   ensure oxmysql
   ensure ox_lib
   ensure nex_garages
   ```

5. Restart your server. You should see garage blips on the map and `Framework initialized and ready!` in the console.

## Updating

Your `config.lua` is preserved when you replace the resource — your settings won't be wiped.

1. Back up your database (always a good idea).
2. Replace the resource files with the new version.
3. Re-import the SQL file only if the update notes mention schema changes.
4. Restart the server.
