## The default ox_lib UI still shows up

The most common causes:

1. **`nexdev_uipack` isn't running.** From F8:

   ```
   lua print(GetResourceState('nexdev_uipack'))
   ```

   It should print `started`. If it prints `missing` or `stopped`, fix `server.cfg` and restart.

2. **You're running upstream `ox_lib`, not our fork.** Only our fork at [github.com/nex-dev-org/ox_lib](https://github.com/nex-dev-org/ox_lib) activates the redesigned UI.

3. **Wrong load order.** `ox_lib` must start **before** `nexdev_uipack`:

   ```
   ensure ox_lib
   ensure nexdev_uipack
   ```

## Folder is named ox_lib-main

GitHub's ZIP download adds the branch name to the folder. Rename it to exactly `ox_lib` before starting the server — the resource name must match.

## I have two ox_lib folders

Don't. Run a single `ox_lib` resource (the fork). Two resources with the same name will give you unpredictable behaviour.

## I stopped nexdev_uipack and now there's no UI at all

`ox_lib` should fall back to its default UI on the next call. If it doesn't, restart `ox_lib` and try again.
